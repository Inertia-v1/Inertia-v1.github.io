# 文章时间不显示问题修复

## 问题

配置了 `displayUpdatedDate = true` 和 `enableGitInfo = true`，但文章底部不显示时间。

## 原因

TOML 配置结构错误：

```toml
[params]
  [params.footer]
    enable = false

  displayUpdatedDate = true  # 被解析为 params.footer.displayUpdatedDate
```

`displayUpdatedDate` 位于 `[params.footer]` 之后，被解析为子段属性。

## 修复

将字段移到 `[params]` 段首部：

```toml
[params]
  displayUpdatedDate = true
  dateFormat = '2006年1月2日'

  [params.navbar]
    # ...
  [params.footer]
    # ...
```

## 时间显示条件

| 方式 | 配置 |
|------|------|
| Git 自动获取 | `enableGitInfo = true` + 文件已提交 |
| 手动指定 | front matter 添加 `date` / `lastmod` |

### Git 自动获取的前提条件

启用 `enableGitInfo = true` 后，Git 自动获取文章时间：

1. md 源文件必须被 Git 跟踪（`git add`）
2. 必须已提交（`git commit`）
3. 部署时需重新构建（非纯静态部署）

未跟踪或未提交的文件没有 Git 历史，`.Lastmod` 无法获取时间。纯静态部署（只上传 public 目录）不触发 Git 信息获取。

### 手动指定示例

```yaml
---
date: 2025-05-17
lastmod: 2025-05-18
---
```
