# 复制按钮跨域失败修复

## 现象

页面右上角"复制 Markdown"按钮点击无反应，浏览器控制台报 `Failed to fetch` 错误。

## 原因

Hextra 的复制按钮通过 `fetch(button.dataset.url)` 获取 Markdown 源文件。`data-url` 由 Hugo 的 `baseURL` 生成，本地开发时指向 `https://inertia-v1.github.io/`，而服务运行在 `localhost`，请求跨域 404。

## 修复

将 `hugo.toml` 中的 `baseURL` 改为相对路径：

```toml
baseURL = '/'
```

本地和生产环境均可正确解析，复制按钮正常工作。

> **注意**：使用相对路径时，`hugo server` 无需额外指定 `--baseURL`。
